Live Location updates

Accessing GeoLocation with navigator.geolocation.

The Geolocation API is accessed via a call to navigator.geolocation; this will cause the user's browser to ask them for permission to access their location data. If they accept, then the browser will use the best available functionality on the device to access this information (for example, GPS).

This will featch the Latitude and Longitude of the user which will be used to get AQI Report at their Lat and Long!

Ne Database

NeDB is a lightweight database written entirely in Javascript, and that implements the well-known MongoDB API.

We can think of it as SQLite for MongoDB projects

The data that is fetched for each user is stored in a database and summary of each entry is stored in another database for future project based on Python Visualization

API

An application program interface (API) is a set of routines, protocols, and tools for building software applications. Basically, an API specifies how software components should interact.

List of API we have used to featch data.

Openweathermap.org Openaq.org Visualcrossing.com Locationiq.com

Openweathermap API : It is used to featch the location weather like temprature, humidity, couldiness.

Openaq API : It is used to featch the location parameters like NO2, SO2, CO, PM2.5, PM10 etc.

Visualcrossing API : It is used to featch the location weather summary.

Locationiq API : It is used to featch the location of sensors and city/area name.
